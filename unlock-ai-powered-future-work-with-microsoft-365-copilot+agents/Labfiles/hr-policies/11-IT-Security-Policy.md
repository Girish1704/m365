# CONTOSO CORPORATION

## IT SECURITY AND ACCEPTABLE USE POLICY

**Document Number:** HR-POL-011  
**Effective Date:** January 1, 2025  
**Last Reviewed:** December 5, 2024  
**Policy Owner:** Information Technology Department  
**Approved By:** David Kim, Chief Information Security Officer

---

### 1. PURPOSE

This policy establishes guidelines for the acceptable use of Contoso Corporation's information technology resources and outlines security requirements to protect company data, systems, and networks.

---

### 2. SCOPE

This policy applies to:
- All employees, contractors, and temporary workers
- All company-owned devices (computers, phones, tablets)
- Personal devices used for work (BYOD)
- All company networks, systems, and data
- Cloud services and applications
- Remote and on-site work environments

---

### 3. ACCEPTABLE USE

#### 3.1 Permitted Uses

Company IT resources are provided for business purposes. Limited personal use is permitted provided it:
- Does not interfere with work duties
- Does not consume excessive bandwidth or storage
- Complies with all company policies
- Does not create legal liability

#### 3.2 Prohibited Activities

The following activities are strictly prohibited:

| Category | Prohibited Actions |
|----------|-------------------|
| **Illegal Activity** | Hacking, piracy, fraud, illegal downloads |
| **Malicious Actions** | Spreading malware, unauthorized access, data theft |
| **Inappropriate Content** | Pornography, hate speech, violent content |
| **Harassment** | Bullying, threatening, discriminatory messages |
| **Commercial Use** | Running personal businesses, unauthorized sales |
| **Gambling** | Online gambling or betting sites |
| **Data Breach** | Unauthorized sharing of confidential information |

---

### 4. PASSWORD AND AUTHENTICATION

#### 4.1 Password Requirements

| Requirement | Standard |
|-------------|----------|
| Minimum length | 12 characters |
| Complexity | Upper, lower, number, special character |
| Expiration | 90 days |
| History | Cannot reuse last 12 passwords |
| Lockout | 5 failed attempts |

#### 4.2 Multi-Factor Authentication (MFA)

MFA is required for:
- All remote access
- Email access from new devices
- VPN connections
- Admin/privileged accounts
- Cloud applications

#### 4.3 Password Best Practices

- Use unique passwords for each system
- Use a password manager (LastPass provided)
- Never share passwords with anyone
- Never write passwords on sticky notes
- Change passwords immediately if compromised
- Report suspected password theft to IT Security

---

### 5. EMAIL AND COMMUNICATIONS

#### 5.1 Email Guidelines

| Requirement | Description |
|-------------|-------------|
| Business use | Email is for business communications |
| Confidentiality | Do not send sensitive data unencrypted |
| Retention | Follow records retention schedule |
| Personal use | Minimal personal email is permitted |

#### 5.2 Email Security

- Do not open suspicious attachments
- Verify sender identity before clicking links
- Report phishing attempts to security@contoso.com
- Do not use email for sensitive data transfer
- Use encryption for confidential communications

#### 5.3 Prohibited Email Activities

- Sending spam or chain letters
- Impersonating others
- Sending confidential data externally without encryption
- Using personal email for business
- Auto-forwarding to personal accounts

---

### 6. INTERNET AND WEB ACCESS

#### 6.1 Monitoring

Contoso monitors internet usage to ensure:
- Policy compliance
- Network security
- Bandwidth management
- Legal compliance

**Employees should have no expectation of privacy when using company resources.**

#### 6.2 Blocked Categories

The following website categories are blocked:

- Adult/explicit content
- Gambling
- Malware/phishing sites
- Peer-to-peer file sharing
- Proxy/anonymizer sites
- Weapons
- Hate/violence

#### 6.3 Social Media

| Platform | Policy |
|----------|--------|
| LinkedIn | Permitted for professional use |
| Other social media | Limited personal use during breaks |
| Company social media | Only authorized personnel |

When posting about work on personal accounts:
- Do not share confidential information
- Clearly state views are your own
- Do not speak on behalf of Contoso
- Follow the Social Media Guidelines

---

### 7. DEVICE SECURITY

#### 7.1 Company Devices

| Requirement | Description |
|-------------|-------------|
| Encryption | All devices must be encrypted |
| Updates | Install security updates within 48 hours |
| Antivirus | Must have approved security software |
| Lock screen | Auto-lock after 5 minutes of inactivity |
| Physical security | Never leave devices unattended |

#### 7.2 Mobile Devices

- Enroll in Mobile Device Management (MDM)
- Enable device PIN/biometric lock
- Enable remote wipe capability
- Do not jailbreak or root devices
- Report lost/stolen devices immediately

#### 7.3 Bring Your Own Device (BYOD)

Personal devices used for work must:
- Be enrolled in MDM
- Meet minimum security requirements
- Have work data in separate container
- Be available for remote wipe of work data
- Have updated operating system

---

### 8. DATA CLASSIFICATION AND HANDLING

#### 8.1 Data Classification Levels

| Classification | Description | Examples |
|---------------|-------------|----------|
| **Public** | No restrictions | Marketing materials, press releases |
| **Internal** | Company use only | Policies, org charts, internal memos |
| **Confidential** | Restricted access | Customer data, financial reports, HR records |
| **Restricted** | Highly sensitive | Trade secrets, M&A data, passwords |

#### 8.2 Handling Requirements

| Classification | Storage | Transmission | Disposal |
|---------------|---------|--------------|----------|
| Public | Any | Any | Standard delete |
| Internal | Approved systems | Internal email OK | Standard delete |
| Confidential | Encrypted systems | Encrypted only | Secure delete |
| Restricted | Encrypted + access control | Encrypted only | Certified destruction |

#### 8.3 Data Loss Prevention

- Do not copy confidential data to personal devices
- Do not upload company data to personal cloud storage
- Do not send confidential data to personal email
- Use approved file sharing methods only
- Report data breaches immediately

---

### 9. NETWORK SECURITY

#### 9.1 Network Access

| Access Type | Requirements |
|-------------|-------------|
| Office network | Automatic with company devices |
| Guest network | For visitors only, no internal access |
| VPN | Required for remote access to internal systems |
| Wi-Fi | Use only approved networks |

#### 9.2 Remote Access

- Use VPN for all internal system access
- Never use public Wi-Fi without VPN
- Ensure home networks are secure
- Log off when session is complete

#### 9.3 Prohibited Network Activities

- Connecting unauthorized devices to network
- Running network scanning tools
- Attempting to access unauthorized systems
- Sharing network credentials
- Installing unauthorized network equipment

---

### 10. SOFTWARE AND APPLICATIONS

#### 10.1 Approved Software

- Only install software from IT-approved sources
- Software installations are logged and monitored
- Unauthorized software will be removed

#### 10.2 Prohibited Software

- Pirated or unlicensed software
- Hacking or security testing tools
- File sharing/torrent applications
- Cryptocurrency mining software
- Games (unless work-related)

#### 10.3 Cloud Services

| Status | Service Type |
|--------|-------------|
| **Approved** | Microsoft 365, Salesforce, approved vendors |
| **Request Required** | New cloud services require IT review |
| **Prohibited** | Personal cloud storage for work data |

---

### 11. PHYSICAL SECURITY

#### 11.1 Workstation Security

- Lock computer when leaving desk (Win+L)
- Secure sensitive documents
- Use privacy screens in public areas
- Clear desk at end of day

#### 11.2 Secure Areas

- Data centers require authorized access
- Server rooms are logged and monitored
- Do not tailgate through secure doors
- Report unauthorized access attempts

---

### 12. INCIDENT REPORTING

#### 12.1 What to Report

Report to IT Security immediately:
- Lost or stolen devices
- Suspected malware or virus
- Phishing emails or suspicious messages
- Unauthorized access attempts
- Data breaches or leaks
- Security vulnerabilities

#### 12.2 How to Report

| Method | Contact |
|--------|---------|
| Email | security@contoso.com |
| Phone | +1 (555) 123-4599 (24/7) |
| Portal | security.contoso.com |
| Emergency | IT Help Desk x4599 |

#### 12.3 Response Time

All security incidents are prioritized:

| Priority | Response Time | Example |
|----------|---------------|---------|
| Critical | Immediate | Active breach, ransomware |
| High | Within 1 hour | Lost device with data, phishing success |
| Medium | Within 4 hours | Suspicious activity, policy violation |
| Low | Within 24 hours | Security questions, guidance needed |
